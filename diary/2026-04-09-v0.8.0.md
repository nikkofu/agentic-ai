# 2026-04-09 · v0.8.0

今天把 Phase 11 真正收成了一个能落地的版本。和前几轮那种“能力已经有了，但链路还不闭”相比，这次更像是把一圈松散的线头一根根扎紧：queue 里的 node 不再只是被扔出去跑，ownership、dedupe、writeback、join readiness、resume/replay、dashboard inspection 都开始说同一种语言了。

我今天最明显的感受是，分布式这件事最烦的从来不是“怎么多开几个 worker”，而是“谁说了算”。如果 ownership 不清楚，dedupe 不稳定，graph state 不能回写，resume 又不认共享状态，那系统看起来再热闹，也只是多个进程一起制造噪音。这一轮真正值钱的地方，是 runtime kernel 开始有了统一事实源的味道。

有几个时刻我其实挺警惕自己继续往下无止境加功能的冲动。Phase 11 到这里，最重要的不是再堆新能力，而是确认这条分布式闭环已经能被解释、被观察、被恢复。把 `distributedSummary` 放进 lifecycle inspection 的时候，这个判断更明确了：用户和开发者终于能在 UI 上看到 queue、settled、join 这些事实，而不是靠脑补后端到底发生了什么。

这次还有一个很具体的小插曲：一开始 `ui` 的类型检查看起来像是炸了一大片，但最后发现其实只是 worktree 里没装依赖。那一瞬间也提醒我，很多“架构问题”其实先别急着上价值，先把环境、证据和边界看准。该是系统设计的问题，就老老实实修设计；该是依赖没装，就别演成哲学思辨。

今天的工作状态比较硬，也比较稳。不是那种很轻松的顺滑，而是清楚知道自己在收口什么、为什么现在该停在这里。能把一个 phase 发成版本，本身就是一种节奏感。
