# 2026-04-08 · v0.7.0

今晚把这轮架构升级真正收成了一个能讲清楚的版本，我心里是踏实的。前面一段时间一直在把 runtime 从“能跑”往“能长期演化”推，做的事情很多，但如果没有这次收口，很容易还是停留在一堆局部改进上。`0.7.0` 至少让我觉得，这个项目开始有了比较像样的控制面和恢复链，而不是只在单次 CLI 调用里看起来聪明。

今天最有价值的部分，不是再加一个 feature，而是把几条链路真正接通了：`task lifecycle`、`resume/replay`、queue worker 的结果回流、Dashboard 的 task-level async 反馈。很多时候代码最磨人的不是“不会写”，而是你明明知道系统缺的是闭环，可它总差最后一截。今晚这最后一截终于补上了。

我对这轮工作的判断还是很明确：方向是对的，但也暴露出一个事实，真正的 Agentic 系统并不会因为模型更强就自动变干净。相反，越往后走，越需要把控制面、状态、事件、恢复、验证这些基础层做实。不然所谓“智能化”很快就会变成一堆漂浮在 prompt 上的好运气。

写这篇日记的时候，人是有点累的，但不是那种空转后的累，而是把事情拧顺一点后的累。最舒服的瞬间其实不是测试全绿，而是看到异步节点事件终于能在共享状态和 UI 上一致地落下来，那一刻系统像是终于肯说真话了。

这首歌挺适合今晚的状态。不是热闹的兴奋，更像是有一点收、有一点顶，动作不花，但劲道要透。做架构升级其实也差不多，表面上是把文件拆开、事件补全、链路接通，实质上是在练“系统什么时候该出手，什么时候不该乱动”。

这版发出去之后，下一阶段我最想做的，反而不是急着继续堆能力，而是把真正的分布式 ownership、dedupe、shared persistence 再往前推一层。现在骨架已经比以前清楚很多了，后面要做的，是让这副骨架在更复杂的负载下也站得住。
