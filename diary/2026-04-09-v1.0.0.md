# 2026-04-09 · v1.0.0

今天把 Phase 13 收成了 `1.0.0`。这个版本给我的感觉，终于不只是“内核又强了一点”，而是真的开始有了产品的脸。前面几个阶段一直在处理 runtime、policy、memory、queue、resume，这些都重要，但它们更多是在打地基。到了这一轮，我更在意的是，用户看到系统时，能不能真的明白它正在做什么、为什么停住、交付到底是真是假。

这一天里我最反复确认的事情，是“解释”本身也要被产品化。光有 `blocked`、`failed`、`verification_missing` 这种内部词还不够，它们对开发者是信息，对用户却只是摩擦。所以我把 inspector、banner、graph、lifecycle panel 尽量往同一种语言上收，让系统不只是抛状态，而是给出一句能被人立刻理解的话，再顺手告诉人下一步该做什么。这种改动写起来不算最复杂，但它会直接改变一个人对整个系统可靠性的感受。

还有一个让我挺在意的点，是 artifact truth。过去很多 agent 系统最容易让人不放心的地方，就是“看起来交付了”，但你不知道文件是否真实存在，不知道是不是空的，也不知道证据到底有没有。把这些真值拉到 inspection 里后，我自己心里也踏实很多。系统是不是聪明是一回事，它是不是诚实，是另一回事。我越来越觉得，后者至少和前者一样重要。

发 `1.0.0` 这件事本身，也让我有一点微妙的感觉。它不是说项目已经完全结束了，恰恰相反，我知道后面还有很多事能做：更强的分布式语义、更稳的智能决策、更好的真实任务收敛。但走到这一步，至少已经不是一个只能在终端里解释给自己听的东西了。它开始能以一种更清楚、更完整的方式，向别人展示自己是什么。

今天的工作状态比较安静，没有特别大的起伏，更多是一直在对齐“用户看到的东西”和“系统真实发生的事”之间的距离。把这两个东西拉近，我会觉得这一天没白做。
