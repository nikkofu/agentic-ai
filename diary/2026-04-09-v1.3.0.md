# 2026-04-09 v1.3.0

今天把 Phase 16 做完了。最重要的不是“多加了一层 memory”，而是终于把 memory 从 runtime 的隐形底座，往产品能力推进了一步。`personal / project / task` 的边界清楚之后，很多东西一下子不再混了：什么该长期记住，什么只该作为任务过程存在，什么又该变成 Dream 的反思材料，终于有了稳定的落点。

Dream 这块我刻意压住了冲动。它很容易被做成“看起来很自主”，但实际上偷偷做了一堆不可控的事。现在把它限制成反思、压缩、技能草稿和建议生成，是更对的路径。先让它会想、会总结、会积累，再谈会不会自己去行动。

这次最有意思的一个小坑，是 memory entry 写进去了，但 task 过滤还是拿不到。最后发现不是 executor 没写，而是 markdown 解析把 `taskId` 和 `source_refs` 吃掉了。修完之后，整条链路才算真的打通。很多阶段都在重复这个感受：真正的难点不在“有没有功能名”，而在一条事实链是否完整。
